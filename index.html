🔄 Complete Processing Pipeline

1

Input Reception

User submits YouTube URL, PDF, DOCX, or pasted text via React interface.

2

Text Extraction

YouTube: YouTubeTranscriptAPI or Whisper • PDF: PyPDF2 • DOCX: python-docx

3

Embedding Compression ⭐ NEW

all-MiniLM-L6-v2 embeddings compress long content to 2,000 focused words before LLM call (3-4× speed improvement)

4

Four-Tier Summarization

Groq LLaMA → Gemini → Fine-tuned BART → MMR Extractive (never fails)

5

Output Generation

English summary returned directly. Roman Urdu: LLM generates or Google Translate + 16,069-entry dictionary normalization

6

About This Project

Digest-U/H is a Final Year Project (FYP) for BS Computer Science (2025-2026) designed to solve a real problem: Pakistani university students struggle to quickly summarize large volumes of educational content and create study materials.

The project combines React frontend, Flask backend, and a sophisticated 4-tier AI pipeline that gracefully degrades from cutting-edge LLMs to offline extractive methods. Every single design decision prioritizes reliability—the system never completely fails.

Key innovation: Embedding-based speed optimization (v4) that compresses long transcripts using semantic similarity before LLM processing, achieving 3-4× faster responses on long content.

🎓 Educational Focus
Designed specifically for Pakistani university students. Supports Roman Urdu output and culturally relevant educational content.
🔄 Reliability First
4-tier fallback system ensures the app always works, even if internet/APIs fail. MMR extractive method provides offline summarization.
⚡ Performance
Embedding compression reduces LLM input by 10× for long content. Smart caching and batch processing optimize every request.
📊 Rigorous Evaluation
Automated ROUGE + BLEU scoring on 98-example dataset. Every search evaluated automatically. Results stored in database.
